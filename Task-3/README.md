# Решение задачи №3

---

## Как работает отправка пушей
- Пользователь разрешает пуши
- Создается токен для пушей, который привязывается к конкретному пользователю. По аналогии с моим проектом - к id пользователя.
- Токен хранится на бэке и вызывается, когда в одном из сервисов (корзина, страница заказа и т.д.) возникает событие требующее отправить пуш.
- Сервис уведомлений получает информацию о событии и формирует текст пуша.
- **Важная отметка**: backend не разнимается отправкой пушей, он создает задачу для стороннего сервиса отправки пушей - FCM для Android и APNs для iOS.
